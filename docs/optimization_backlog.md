# Agent 正确性优化清单

来源：`agent_data_analysis_eval_plan.md` 中的真实性测试题库与改进建议。

## 本轮已落地

- 数据粒度识别：画像阶段自动识别订单流水、客户-时间快照、候选粒度字段、时间字段、时间范围和重复粒度记录。
- 指标口径提示：新增轻量业务指标注册表，覆盖 MRR、销售额、利润、利润率、流失率、复购率、NPS、CSAT、逾期金额。
- 报告自解释：Markdown 报告增加业务粒度、候选粒度字段、时间范围和指标口径提示。
- 数据质量独立扫描增强：不只读取标记字段，新增业务键重复、0 金额、负利润、销售额/利润/利润率公式一致性检查。
- 质量明细可追溯：数据质量结果增加 `data_quality_detail_rows` 明细表，前端结果区改为展示全部结果表。
- 回归保护：新增粒度识别和业务质量规则测试。
- SaaS 口径与风险客户问题：新增当前 MRR vs 累计 MRR 口径对比、高风险客户 MRR 排名两条固定分析路径，并加入 `agent_eval`。
- 业务模板路由：新增统一 `business_template_analysis`，覆盖账款与续约风险、客户成功优先级、渠道风险、行业市场、分层/套餐、扩张收缩、健康信号、销售总览、订单状态、商品 Pareto、折扣影响、支付结构、销售渠道策略。
- 企业级评测修复：新增 `agent_eval/enterprise_business_eval.py`，修复未来/离群月份、显式起止月份、数据质量优先级、健康信号 pairwise 相关性、应收金额别名、CRM Pipeline 模板，企业评测从 1/8 提升到 8/8。
- 复合问题处理：新增多问句拆分、重复操作去重和 `multi_analysis` 聚合执行，支持一次提问中同时回答数据概览、Top 查询、数据质量、业务模板等多个子问题。

## 下一批优先级

1. 增加多 sheet 支持，优先读取 `Dictionary` 工作表解释字段，并支持 `Data` 与 `Customers` 关联。
2. 扩展中文问题解析，支持“只看已完成”“客单价”“贡献率”“占比”“环比”等更多组合口径。
3. 增加更多真实业务模板：
   - SaaS：MRR 趋势、流失、扩张/收缩、续约风险、账款风险、客户成功优先级。
   - 销售：经营总览、订单状态、商品 Pareto、渠道效率、折扣影响、区域-品类扩张。
4. 将 S01-S12、O01-O15 拆成可自动评测的回归用例；数值题使用容差断言，TopN 题断言关键行，报告题断言口径关键词。
5. 对失败用例建立闭环：先加入 `agent_eval/cases.json`，再修复解析或工具逻辑，防止回退。
